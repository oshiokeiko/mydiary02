# 📔 My Diary（初心者向け・スッキリ版）

## どんなアプリ？

ブラウザだけで動く、シンプルな日記アプリです。
書いた日記はブラウザの中（ローカルストレージ）に保存されるので、リロードしても消えません。
サーバーは不要で、`index.html` をダブルクリックするだけで使えます。

**学習課題として使っている技術：**

- JavaScript（配列・オブジェクト・繰り返し・条件分岐）
- jQuery（画面の操作・ボタンの反応）
- ローカルストレージ（保存・削除）
- JSON（`JSON.stringify` / `JSON.parse`）
- Tailwind CSS（見た目）

---

## できること

- **日記を書く**：タイトル・気分の絵文字・本文を入力して追加
- **一覧表示**：新しい順に並ぶ
- **削除**：各カードの「削除」ボタンで1件ずつ消せる
- **保存**：ローカルストレージに自動保存（閉じても消えない）

## 工夫した点

### まず PRD（要求仕様書）を作って方針を固めた
いきなりコードを書き始めるのではなく、「習ったこと（配列・オブジェクト・繰り返し・ローカル保存）を反映した日記アプリ」として、最初に **PRD（[PRD.md](PRD.md)）** を作成しました。何を作るか・何は作らないかをはっきりさせてから、方針をチェックして進めました。

### PRD をもとに実装計画を立てた
固めた方針をもとに、**実装の手順（どの機能から、どの順で作るか）** を計画してから着手しました。行き当たりばったりにせず、段階的に組み立てる進め方を意識しています。

### JavaScript を別ファイルに分けた
画面（HTML）と動き（JavaScript）を **別ファイル**（`index.html` と `app.js`）に分けました。役割ごとにファイルが分かれることで、どこに何が書いてあるかが見通しやすく、読み解きやすくなっています。

## ファイル構成

| ファイル | 役割 |
|----------|------|
| `index.html` | 完成版の画面（見た目） |
| `app.js` | 完成版の動き（JavaScript） |
| `style.css` | Tailwindで補えない飾り（全ファイル共通） |
| `steps/step1〜3` | 段階的に作るための学習用ファイル |
| `LEARN.md` | 読み解きガイド |

## 使用技術

| 項目 | 技術 |
|------|------|
| マークアップ | HTML5 |
| スタイリング | Tailwind CSS（CDN）+ カスタムCSS |
| スクリプト | JavaScript + jQuery 3.7.1（CDN） |
| データ保存 | localStorage（JSON形式） |
| ビルドツール | なし |
