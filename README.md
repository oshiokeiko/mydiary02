# 📔 My Diary

## プロダクトの紹介

ブラウザだけで動く、シンプルなローカル日記アプリです。  
書いた日記はブラウザの「ローカルストレージ」に自動保存されるため、ページをリロードしても消えません。  
サーバーや外部サービスへの接続は不要で、`index.html` をダブルクリックするだけで即座に使えます。

**学習課題として取り組んだ技術：**

- JavaScript（条件分岐・配列・オブジェクト・繰り返し）
- jQuery（DOM操作・イベント処理）
- ローカルストレージ（保存・削除）
- JSON（`JSON.stringify` / `JSON.parse`）
- Tailwind CSS（レスポンシブデザイン）

---

## アプリの特徴

- **日記の新規作成・編集・削除**  
  モーダルウィンドウから日記を書いて保存できます。編集・削除もワンクリックで操作できます。

- **気分の絵文字セレクター**  
  😊 😢 😡 😴 🥰 😤 🤔 の7種類から今日の気分を選べます。

- **本文プレビュー表示**  
  一覧カードに本文の冒頭80文字が表示されるので、内容をパッとつかめます。

- **新しい順の自動並び替え**  
  一覧は常に作成日時の新しい順で表示されます。

- **レスポンシブ対応**  
  PC（768px以上）では2カラム、スマートフォンでは1カラムに自動で切り替わります。

- **データの永続化**  
  ページを閉じても・リロードしても、ブラウザのローカルストレージにJSON形式で保存されるため日記は消えません。

## 工夫した点・こだわった点

### `closeModal()` に全リセット処理をまとめた

モーダルを閉じる処理を1つの関数（`closeModal()`）に集約し、そこで以下をまとめてリセットするようにしました。

- 編集中の日記IDを管理する `editingId` を `null` に戻す
- 絵文字の選択状態をリセットする

「キャンセル」「保存する」「×ボタン」「背景の幕クリック」と、モーダルを閉じるきっかけが複数あるため、どこかにリセット漏れが発生しないよう、必ず1つの関数を通る設計にしました。

### HTMLインジェクション対策（`escapeHtml` 関数）

ユーザーが入力したテキストをそのままHTMLに埋め込むと、`<script>` などの危険なコードが実行されてしまう可能性があります。これを防ぐための `escapeHtml()` 関数を実装し、カード描画時に必ずエスケープ処理を通すようにしました。

### イベント委譲（Event Delegation）でカードのボタンに対応

「編集」「削除」ボタンはページ読み込み後に JavaScript で動的に生成されます。動的な要素にはそのままイベントを設定できないため、`$(document).on('click', '.edit-btn', ...)` という書き方（イベント委譲）を使って対応しました。

### Zenn風のダークテーマUI

Zennのデザインを参考に、紺色の背景・白文字・角丸カードのUIをTailwind CSSで実装しました。カードへのホバー時の浮き上がりアニメーションやモーダルの登場アニメーションも加え、使い心地にこだわりました。

## 苦戦した点・共有したいハマりポイント

### `editingId` のリセット漏れによる上書きバグ

新規作成と編集で同じ「保存する」ボタンを共用する設計のため、「どちらのモードか」を `editingId` という変数で管理しています。  
最初の計画では「モーダルを閉じるときにリセットする」とだけ書いており、キャンセル・保存・×ボタンのどれかでリセット漏れが起きると、**新規作成のつもりが前の編集IDを引き継いで既存の日記を上書きしてしまう**バグにつながります。  

対策として `closeModal()` にリセットを一本化し、すべての「閉じる動作」がこの関数を通るよう統一しました。

### 動的に追加した要素にイベントが効かない

`$('.delete-btn').on('click', ...)` のように書いても、**ページ読み込み時点でまだ存在しない要素（後から追加されるカードのボタン）にはイベントが設定されない**という落とし穴がありました。  

解決策は `$(document).on('click', '.delete-btn', ...)` のように、最初から存在する親要素（`document`）に対してイベントを設定し、子要素に委譲する「イベント委譲」を使うことです。

### `data-id` が文字列として取れる

`$(this).data('id')` で取得したIDは **文字列型**で返ってきます。日記データの `id` は `Date.now()`（数値型）で作っているため、`===` で比較すると一致せず、編集・削除が動かないことがありました。  
`Number($(this).data('id'))` で数値に変換することで解決しました。
