# 📋 My Diary PRD（プロダクト要求仕様書）

---

## 1. 概要

ブラウザだけで動く、シンプルな日記アプリ。
日記はブラウザの中（ローカルストレージ）に保存され、リロードしても消えない。
サーバー不要で、`index.html` を開くだけで使える。

---

## 2. 背景・目的

- JavaScript の基礎（配列・オブジェクト・繰り返し・ローカル保存）を、
  実際に動くアプリを通して身につける。

---

## 3. スコープ（やること・やらないこと）

### ✅ やること（この版に入れる機能）
- 日記を書く（タイトル・気分の絵文字・本文）
- 一覧で表示する（新しい順）
- 日記を削除する
- ブラウザに自動保存する（閉じても消えない）

### ❌ やらないこと（あえて入れない）
- 編集機能（「新規」と「編集」の切り替えが初心者には難所のため）
- ポップアップ画面（モーダル）→ 入力フォームは常に表示する
- ログイン／複数ユーザー／他人との共有
- サーバーやデータベースへの保存（ブラウザ内だけで完結）
- 検索・並べ替えの切り替え・カテゴリ分け

---

## 4. 機能要件（くわしい仕様）

### 4-1. 日記を書く
- 入力項目：**タイトル**（1行）・**気分の絵文字**（😊😢😡😴🥰🤔から1つ）・**本文**（複数行）
- 「追加する」ボタンで保存する。
- タイトルか本文が空のときは、注意メッセージを出して保存しない。
- 保存後、入力欄は空に戻り、気分は初期値（😊）に戻る。

### 4-2. 一覧表示
- 保存した日記をカード形式で並べる。
- 並び順は**新しいものが上**（作成日時の新しい順）。
- 各カードに表示する内容：気分の絵文字／タイトル／本文／作成日（YYYY/MM/DD）／削除ボタン。
- 日記が0件のときは「まだ日記がありません」の案内を出す。

### 4-3. 削除
- 各カードの「削除」ボタンで、その1件だけを消す。
- 削除前に「本当に削除しますか？」と確認する。

### 4-4. 保存（永続化）
- 日記の追加・削除のたびに、ブラウザのローカルストレージへ自動保存する。
- ページをリロードしても・閉じても、日記は残る。

---

## 5. データ構造（日記1件の中身）

日記はオブジェクトとして表し、それを配列にためてローカルストレージに保存する。

```js
{
  id:        1717641600000,                  // 見分ける番号（作成時刻の数字）
  title:     "今日のできごと",                // タイトル
  mood:      "😊",                            // 気分の絵文字
  body:      "良い一日だった。",               // 本文
  createdAt: "2026-06-06T09:00:00.000Z"       // 作成日時
}
```

保存時は、この配列を JSON（文字）に変換してローカルストレージへ入れる。

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| 動作環境 | モダンブラウザ（Chrome / Safari / Edge 等） |
| レスポンシブ | スマホは1列、PC（768px以上）は2列で表示 |
| 保存先 | ブラウザのローカルストレージ（外部送信なし＝プライバシー安心） |
| 表示の安全性 | 入力文字の記号をエスケープし、表示崩れを防ぐ |
| 読みやすさ | コードに日本語コメントを多めに付け、初学者が読める状態を保つ |
